## 2026-05-29 - Argparse Resource Exhaustion Prevention\n**Vulnerability:** The call center simulation accepted unbound large integers for time loops, sleep intervals, and array sizing parameters, creating a Denial of Service (DoS) risk through resource exhaustion.\n**Learning:** In CLI scripts handling threading or loops, relying on type constraints (`type=int`) is insufficient; numerical bounds checking must be enforced at both the parser level and object initialization level.\n**Prevention:** Always validate maximum viable values for loop intervals and thread pool sizes in CLI arg parsing logic to fail fast and securely before memory allocation or blocking operations begin.
